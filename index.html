변경 아카이브

Meta 동기화가 감지한 캠페인·광고세트 이름 변경 이력입니다. 세트·캠페인 ID 기준으로 추적하므로 매체에서 이름을 바꿔도 어떤 세트가 어떻게 바뀌었는지 남습니다. 자동 동기화(자정·정오) 때마다 갱신됩니다.

Meta 생성 · dry-run 미리보기

빌더 광고세트를 골라 Meta에 보낼 캠페인·광고세트 페이로드를 미리 만들어 봅니다. 실제 생성은 하지 않습니다(권한 불필요). 목표·입찰·타게팅 매핑과 부족한 값(예산·픽셀·페이지 등)을 먼저 점검하는 단계입니다.

⚠️ 현재는 미리보기 전용입니다. 실제 캠페인 생성(쓰기)은 ads_management 권한·앱 심사 확보 후 다음 단계에서 연결됩니다. 생성 시에도 항상 PAUSED로 만들어 즉시 노출되지 않게 합니다.
① 캠페인 생성 독립 실행 — 캠페인만 만들기
② 광고세트 생성 대상 캠페인 선택 → 광고세트 정의
③ 광고 소재 세팅 대상 광고세트에 소재 연결
이미지/영상 에셋 업로드는 다음 단계(Phase 3)입니다. 여기선 소재명·UTM·랜딩 기준 페이로드만 미리봅니다.

설정/매핑 · 클룹·스프린트 공통

모든 자동 생성의 단일 기준 데이터이며, 기존 구글시트를 대체합니다. (이 탭은 클룹/스프린트 공통 영역) TAM은 콘텐츠마케터 파트장, 그 외(매체·합성·제품·스킴 등)는 퍼포먼스마케터가 관리합니다. 변경 시 전 탭·전 팀원에게 즉시 반영됩니다.

광고세트명  :  매체_제품코드_캠페인목표_입찰_[최적화]_타게팅_스킴_TAM_날짜
